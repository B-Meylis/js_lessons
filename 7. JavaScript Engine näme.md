## JavaScript Engine näme?

**JS Engine(Движок)** – bu siziň JavaScript koduňyzy okaýan, düşünýän we ýerine ýetirýän programmadyr.

Iň meşhurlary:
1. **V8** — Chrome, Edge, Node.js
2. **SpiderMonkey** — Firefox
3. **JavaScriptCore** — Safari

**Ýönekeý meňzetme**:
Engine aşpez ýalydyr. Siz oňa resept (kod) berýärsiňiz, aşpez ony okaýar, görkezmelere düşünýär we nahary taýýarlaýar (netijäni öndürýär).


**Engine hemişe iki tapgyrda işleýär**:
```text
1-nji TAPGYR: PARSING (okamak + taýýarlyk)
          ↓
2-nji TAPGYR: ÝERINE ÝETIRMEK (setir-setir ýerine ýetirmek)
```


**1-nji tapgyr: Parsing (konteksti döretmek)**

Bu tapgyrda engine:  
1️⃣ Kody başyndan ahyryna çenli okaýar  
2️⃣ Ähli üýtgeýji (variable) we funksiýalary tapýar  
3️⃣ Ýerine ýetiriş kontekstini (Execution Context) döredýär  
4️⃣ Üýtgeýjiler üçin ýat bölüp berýär  

**Bellik**❗️ Kod entek ýerine ýetirilmeýär, diňe taýýarlanýar.
```js
console.log(a); // undefined (Ýalňyşlyk ýok!)
var a = 5;
console.log(a); // 5
//Engine muny takmynan şeýle görýär:
//1-nji tapgyr:
//"Oho, var a bar eken"→ a = undefined döredilýär.

//2-nji tapgyr:
//console.log(a) → undefined
//a = 5
//console.log(a) → 5
```

**2-nji tapgyr: Ýerine ýetirmek (Execution)**  
Indi engine:  
1️⃣ Setirleri tertip boýunça okaýar  
2️⃣ Üýtgeýjilere bahalary berýär  
3️⃣ Funksiýalary çagyrýar  
4️⃣ Operasiýalary ýerine ýetirýär  

**JS Engine konteksti nädip döredýär?**

Her bir Execution Context (ýerine ýetiriş konteksti) döredilende JavaScript Engine 3 ädimden geçýär:
```text
┌─────────────────────────────────────────────┐
│  ÄDIM 1: Üýtgeýjiler gurşawyny döretmek     │
│  ┌────────────────────────────────────┐     │
│  │ • arguments (funksiýalar üçin)     │     │
│  │ • ähli üýtgeýjiler (var, let, const)│    │
│  │ • ähli funksiýalar (function       │     │
│  │   declaration arkaly yglan edilen) │     │
│  └────────────────────────────────────┘     │
│                    ↓                        │
│  ÄDIM 2: Scope zynjyryny döretmek │
│  ┌────────────────────────────────────┐     │
│  │ • parent kontekste salgylanma         │     │
│  │ • (şol [[Scope]] bolýar)           │     │
│  └────────────────────────────────────┘     │
│                    ↓                        │
│  ÄDIM 3: this bahasyny kesgitlemek          │
│  ┌────────────────────────────────────┐     │
│  │ • funksiýanyň nähili çagyrylandygyna│    │
│  │   bagly                            │     │
│  └────────────────────────────────────┘     │
└─────────────────────────────────────────────┘
```
Her ädim barada giňişleýin

📌 **Hoisting** – JavaScript-de üýtgeýjileriň we funksiýalaryň kod ýerine ýetirilmezden ozal scope-yň(область видимости) başyna "göterilen" ýaly hereket edýän mehanizmidir

**Bellik**❗️Göterilýän zat diňe yglan edilenler (declarations) bolup,  bahasy berlenler(initializations) däldir.
Şeýle hem, kod fiziki taýdan hereket etmeýär – diňe ýerine ýetiriş kontekstini döretmek wagtynda bolup geçýän hereketler.
